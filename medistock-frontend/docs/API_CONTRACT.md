# API Contract - MediStock

## Base URL

http://localhost:8080/api

---

# 1. Data Models

## User / Institution

{
  "id": 1,
  "nume": "Spitalul Clinic Judetean Cluj",
  "tip": "HOSPITAL",
  "judet": "Cluj",
  "adresa": "Str. Clinicilor nr. 1",
  "username": "spital_cluj",
  "parola": "not_returned_in_responses"
}

Field rules:
- id: integer, generated by backend
- nume: required
- tip: ADMINISTRATIVE | HOSPITAL | PHARMACY
- judet: required
- adresa: required
- username: required, unique
- parola: required only on create/login, never returned in plaintext

---

## Medication

{
  "id": 1,
  "name": "Paracetamol",
  "category": "Painkiller",
  "stock": 120,
  "expiryDate": "2026-10-10",
  "daysUntilExpiry": 167,
  "batchNumber": "PARA-001",
  "price": 10.5,
  "supplier": "Farmex",
  "status": "OK"
}

Field rules:
- id: integer, generated by backend
- name: required
- category: required
- stock: integer, must be >= 0
- expiryDate: date (YYYY-MM-DD)
- daysUntilExpiry: calculated (expiryDate - current date)
- batchNumber: required, unique
- price: float, must be >= 0
- supplier: required
- status: LOW_STOCK | EXPIRING | EXPIRED | OK (calculated)

# 3. Medications API

## GET /medicaments

Query params:
- hospitalId (optional)
- category (optional)
- status (optional)
- page (default 0)
- size (default 20)

Example:
GET /api/medicaments?hospitalId=12&category=Painkiller&page=0&size=20

Response 200:
{
  "content": [ ... ],
  "page": 0,
  "size": 20,
  "totalElements": 160,
  "totalPages": 8
}

---

# 7. Error Handling Standard

All errors follow this format:

{
  "timestamp": "2026-04-27T18:00:00",
  "status": 400,
  "error": "Bad Request",
  "message": "Stock cannot be negative",
  "path": "/api/medicaments/1/stock"
}

Supported status codes:

- 400 Bad Request → invalid input
- 401 Unauthorized → invalid login / missing token
- 403 Forbidden → no permission
- 404 Not Found → resource not found
- 409 Conflict → duplicate resource
- 500 Internal Server Error → unexpected error
